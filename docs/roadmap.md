## Release Philosophy

### Versioning

We follow semantic versioning:
- **Major (1.0, 2.0):** Breaking changes
- **Minor (0.3, 0.4):** New features, backwards-compatible
- **Patch (0.3.1, 0.3.2):** Bug fixes

### Release Cycle

- **Minor releases:** Quarterly (every 3 months)
- **Patch releases:** As needed (bug fixes)
- **Major releases:** Annually or when warranted

### Backwards Compatibility

We maintain backwards compatibility within major versions:
- **v0.x.x:** May break between minors (pre-1.0 stability)
- **v1.x.x+:** Guaranteed backwards compatibility within majors
