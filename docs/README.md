## Documentation Standards

### Writing Style

- **Clear over clever**: Use plain language
- **Show, don't just tell**: Include code examples
- **Assume beginner mindset**: Don't assume prior knowledge
- **Link generously**: Connect related concepts

### Code Examples

- Always include working code
- Show imports and setup
- Include expected output
- Explain non-obvious parts

### Maintenance

Documentation is part of the codebase:
- Update docs when changing APIs
- Add examples for new features
- Fix errors and unclear sections
- Version docs with code
